Paso

Definir Objetivos


Para el primer paso, debemos empezar definiendo los objetivos que va a tener nuestro manual. Por lo general casi siempre el objetivo principal para los usuarios, suele ser el de responder dudas e inquietudes que pueda tener un usuaio "común".



Paso

Reconocer los requisitos del sistema


En este aparatdo, debemos saber que clase de usuario va a utilizar nuestro software o a que tipo de usuario va a ir dirigido. Para saberlo, necesitamos mirar si nuestro usuario necesita tener algun tipo de conocimiento requerido para utilizar el software.



Paso

Especificaciones tecnicas hardware y software


Para este apartado necesitamos identificar las especificaciones tecnicas del hardware y software que el sistema requiera. Por lo general se deben especificar los requerimientos mínimos que necesita el programa para poder funcionar.
En el apartado del software, debemos explicar si nuestro programa requiere de alguna ayuda externa, como sería una maquina virtual. Tambien debemos aclarar las limitaciones que el programa puede tener.



Paso

Como ingresar al sistema

En este apartado se debe explicar lo más claro y detalladamente como se puede ingresar al sistema. Ademas se debe explicar las formas en que debe ingresar cada tipo de usuario (esto en caso de que tengamos roles para cada yipo de usuario)



Paso

Explicar interfaz grafica

En este apartado, debemos explicar todo lo que tenga que ver con lo visual por ejemplo: se debe explicar la barra de navegacion, losfoprmularios y todo lo que el usuario pueda ver e interactuar



Paso

Especificar mapa de navegacion

En este apartado, debemos presentar de forma visual el recorrido que va hacer el visitante a la hora de entrar a nuestro programa. Por lo general se suele mostrar un mapa conceptual el cual muestra la arquitectura de nuestra página
